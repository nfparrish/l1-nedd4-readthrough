## 1. Background / Why This Branch Exists

The STAR-based MCF7 pipeline produces a maximum depth of ~2 reads in the
**downstream sense window** (paper-style readthrough window), whereas Philippe
et al. (Fig. 3B) shows ~50. Multiple filter hypotheses were ruled out (pooled
controls, relaxed MAPQ, duplicates included — still max depth 5). Conclusion:
the mismatch is either an interval or an aligner difference. This branch
re-maps with **TopHat2**, the aligner used in the paper, to test that
hypothesis.

## 7. If Alignment Failed — Recovery Steps

The most likely failure modes for the align step are:

1. **OOM** — increase `#SBATCH --mem` in `02_align_tophat.sh` from 48G to 64G
2. **Missing FASTQ** — confirm trimmed FASTQs exist:
   ```bash
   ls /work/nfp8/MCF7_ctrl/fastq/*_trimmed_P.fq.gz
   ```
   If missing, set `TOPHAT_USE_TRIMMED=0` in `config_test_mcf7_tophat.sh` and resubmit
3. **TopHat2 itself crashes** — check stderr for a stack trace; if the
   Bowtie2 index is corrupt, rebuild it:
   ```bash
   sbatch /hpc/home/nfp8/copilot/2026_03_03/test_mcf7/01_build_bowtie2_index_tophat.sh
   ```

To resubmit the full chain from scratch:
```bash
cd /hpc/home/nfp8/copilot/2026_03_03/test_mcf7
bash run_test_mcf7_tophat.sh
```

To resubmit only align → post → collect → visualize (insert-size env already
good):
```bash
source config_test_mcf7_tophat.sh
JOB_ALIGN=$(sbatch --export=ALL 02_align_tophat.sh | awk '{print $4}')
JOB_POST=$(sbatch --dependency=afterok:${JOB_ALIGN} --export=ALL \
    --array=1-4 ../scripts/03_post_align.sh | awk '{print $4}')
JOB_COLLECT=$(sbatch --dependency=afterok:${JOB_POST} --export=ALL \
    ../scripts/04_collect_results.sh | awk '{print $4}')
JOB_VIS=$(sbatch --dependency=afterok:${JOB_COLLECT} --export=ALL \
    ../scripts/05_visualize.sh | awk '{print $4}')
```
